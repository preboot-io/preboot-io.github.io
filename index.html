Why PreBoot?

Stop reimplementing the same patterns. Focus on your business logic while we take care of the SaaS infrastructure.

Multi-Tenant Authentication

Complete multi-tenant user management with JWT support, role-based access control, and automatic tenant context - all with minimal configuration.

Secure Data Access

A simple @Tenant annotation provides automatic tenant isolation on all your queries. Build complex, type-safe dynamic queries with built-in security filtering.

Observability Ready

Built-in monitoring for slow method calls, detailed error tracking, and health checks. Easily identify performance bottlenecks even in development.

Event-Driven Architecture

Complete event system with synchronous and asynchronous handling, persistent task queues with retry logic, and type-safe events for building reactive applications.

Full-Stack SaaS Solution

Backend and frontend libraries optimized for SaaS development with React components for auth flows, data tables, and form handling - all integrated with tenant context.

Modular Architecture

Pick and choose from modular components that work together seamlessly. Use just what you need with minimal dependencies, or adopt the entire framework for maximum productivity.

How It Works

Getting started with PreBoot is simple. Follow these steps to accelerate your SaaS development.

1
Install the Dependencies
Add PreBoot to your Spring Boot and React projects using Maven/Gradle and npm. Our modules are lightweight and designed to integrate with your existing technology stack.
2
Configure Multi-Tenancy
Set up tenant isolation with just a few annotations. Choose from multiple strategies like schema-per-tenant or discriminator column based on your business requirements.
3
Build Your Business Logic
Focus on what makes your application unique by creating feature-first modules. Use our security-aware repositories and UI components to build CRUD flows in minutes.
4
Scale With Confidence
As your user base grows, leverage our event system, task queues, and observability tools to maintain performance and reliability without massive architecture changes. 
// Entity with multi-tenant support and access control
@Table("documents")
@SecureAccess(
    read = @AccessRule(roles = {"VIEWER", "EDITOR", "ADMIN"}),
    write = @AccessRule(roles = {"EDITOR", "ADMIN"})
)
public class Document {
    @Id
    private Long id;
    
    @Tenant  // Automatic tenant filtering
    private UUID tenantId;
    
    private String title;
    private String content;
    private String status;
    
    @Column("created_at")
    private LocalDateTime createdAt;
    
    @Version
    private Long version;  // Optimistic locking
}
// Secure repository interface
public interface DocumentRepository extends SecureRepository {
    // Find methods automatically filtered by tenant
    Optional findByTitle(String title);
    
    // Custom queries with automatic security
    @Query("SELECT * FROM documents WHERE status = :status AND tenant_id = :#{currentTenant()}")
    List findByStatus(String status);
}

// Repository implementation
@Repository
public class DocumentRepositoryImpl extends SecureRepositoryImpl {
    public DocumentRepositoryImpl(SecureRepositoryContext context) {
        super(context, Document.class);
    }
    
    // Dynamic filtering with automatic tenant security
    public Page search(String titlePattern, String status) {
        SearchParams params = SearchParams.criteria(
            FilterCriteria.like("title", titlePattern + "%"),
            FilterCriteria.eq("status", status)
        )
        .sortBy("createdAt", Sort.Direction.DESC)
        .build();
        
        return findAll(params);  // Tenant filtering applied automatically
    }
}
// Define an event
public record DocumentCreatedEvent(
    UUID documentId,
    UUID tenantId,
    String title
) {}

// Create an event handler
@Component
public class DocumentEventHandler {
    private final NotificationApi notificationApi;
    
    @EventHandler
    public void handleDocumentCreated(DocumentCreatedEvent event) {
        // Send notification email
        EmailParam email = EmailParam.builder(
            "docs@yourapp.com",
            "admin@tenant.com",
            "New Document Created",
            "document-notification")
            .addTemplateParam("documentId", event.documentId())
            .addTemplateParam("title", event.title())
            .build();
            
        notificationApi.sendEmail(email);
    }
}

// Create a task for async processing
TaskPublisher taskPublisher;

@Service
public class DocumentService {
    // ...
    public void processDocument(UUID documentId) {
        // Schedule for async processing with automatic retries
        taskPublisher.publishTask(
            new ProcessDocumentTask(documentId)
        );
    }
}
